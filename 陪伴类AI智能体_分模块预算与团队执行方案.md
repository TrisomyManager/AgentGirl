# 陪伴类 AI 智能体项目分模块预算与团队执行方案

> 版本：V1.0  
> 用途：技术可行性调研后的预算拆解、团队配置、阶段执行与投资评估  
> 口径：人民币 RMB，按中国大陆中小型研发团队项目制估算  
> 说明：本文预算为“研发落地预算模型”，不是外包固定报价。实际费用会受城市、人员资历、是否全职、云模型调用量、美术资产质量、硬件数量、并发规模影响。

---

## 1. 项目目标复述

本项目目标不是普通聊天机器人，而是一个具备以下能力的长期记忆型虚拟陪伴智能体 / 数字生命系统：

1. 稳定人格与 OOC 边界；
2. 长期记忆、短期记忆、用户画像、关系状态；
3. 多模型路由、本地模型实验、云端模型调用；
4. 语音输入、语音输出、情绪语音；
5. Unity / Web / 移动端 / PC 端角色表现；
6. 表情、口型、动作、情绪状态同步；
7. 工具调用、文件/代码/表格能力、安全沙箱；
8. 跨设备注册、心跳、任务下发与状态同步；
9. 监控、日志、链路追踪、数据备份、权限与审计；
10. 后期可扩展动作生成、实时数字人、知识图谱、LoRA 人格实验。

因此，本项目应按“平台型 AI 产品 + 角色表现系统 + 多端协同系统”评估预算，而不是按一个单一 App 或单一 Agent Demo 评估。

- 税前薪资或外包人力成本；

## 3. 分模块预算总览

以下预算按“完整实现文档目标”拆分。每个模块分为三档：

- **基础档**：能跑通核心能力，适合 MVP；
- **产品档**：可用于 Beta 或小规模上线；
- **增强档**：更接近商业化完整版本。

| 模块 | 基础档预算 | 产品档预算 | 增强档预算 | 自研程度 | 优先级 |
|---|---:|---:|---:|---:|---:|
| 1. 产品设计与项目管理 | 20-40 万 | 60-120 万 | 150-250 万 | 中 | P0 |
| 2. 技术架构与基础工程 | 35-70 万 | 100-180 万 | 250-400 万 | 中 | P0 |
| 3. AI 对话编排与模型网关 | 50-100 万 | 120-250 万 | 300-500 万 | 中高 | P0 |
| 4. 人格系统 / OOC / 关系状态 | 40-90 万 | 120-260 万 | 300-600 万 | 高 | P0 |
| 5. 长期记忆系统 | 60-130 万 | 180-380 万 | 450-800 万 | 高 | P0 |
| 6. 语音交互系统 | 40-90 万 | 120-280 万 | 350-700 万 | 中高 | P1 |
| 7. Unity 角色表现系统 | 60-150 万 | 200-500 万 | 600-1200 万 | 高 | P1 |
| 8. Web 管理后台 / 调试台 | 30-70 万 | 100-220 万 | 250-450 万 | 中 | P0 |
| 9. 工具调用与办公能力 | 30-80 万 | 120-300 万 | 350-700 万 | 中高 | P2 |
| 10. 跨设备协同系统 | 40-100 万 | 150-350 万 | 400-900 万 | 高 | P2 |
| 11. 安全、权限、沙箱、审计 | 30-80 万 | 120-300 万 | 350-800 万 | 高 | P1/P2 |
| 12. 持久化、运维、监控 | 30-80 万 | 100-250 万 | 300-700 万 | 中 | P1 |
| 13. 测试、评测与质量体系 | 25-60 万 | 100-240 万 | 300-600 万 | 中高 | P0/P1 |
| 14. 高级数字人 / 动作生成预研 | 30-100 万 | 150-500 万 | 500-1500 万 | 高 | P3 |
| 15. 美术资产与角色内容 | 50-200 万 | 200-800 万 | 800-2000 万 | 高 | P1/P3 |

> 重要判断：如果只做技术 Demo，预算可压缩到 100-200 万；如果要实现文档中的全部平台目标，建议按 800-1800 万区间规划；如果追求高品质数字人、完整多端产品和商业化运营，预算可能进入 2000 万以上。

### 开源/现成可用

- Mem0；
- pgvector；
- Milvus；
- Pinecone；
- Neo4j；
- Redis；

### 需要自研/沉淀

- 什么值得记的判断规则；
- 记忆污染防护；

### 预算风险

## 7. 哪些钱不能省

## 7.1 不能省：技术负责人

没有强架构师，项目会变成各种 API 和 Demo 的拼贴，后期必然返工。

## 7.2 不能省：长期记忆系统

长期记忆是产品壁垒。简单 RAG 不等于陪伴类记忆。

## 7.3 不能省：人格/OOC/关系状态

这个模块决定角色是否像“一个稳定存在的人”。

## 7.4 不能省：Unity 表现基础

如果最终要做数字生命或虚拟陪伴，角色表现不能只停留在网页聊天框。

## 7.5 不能省：测试和管理后台

没有后台和测试集，就无法调人格、调记忆、查 OOC、查设备和工具调用问题。

## 12. 最终预算建议

### 如果你现在还在技术可行性阶段

建议先申请：**100-200 万验证预算**。

用途：

- 核心技术链路验证；
- 对话 + 记忆 + 人格 Demo；
- Unity 角色端初版；
- 语音链路验证；
- 技术风险清单。

### 如果要做可演示 MVP

- 3 个月；
- 做出能给客户/投资人看的完整 Demo。

建议申请：**700-1500 万产品化预算**。

用途：

- 10-12 人团队；
- 6-9 个月；
- 做出可小规模试点、可持续迭代的版本。

### 如果要完整实现文档所有目标

建议规划：**1800-4000 万完整平台预算**。

用途：

- 12-18 个月；
- 12-18 人团队；
- 完整覆盖长期记忆、人格、语音、动作、工具、跨设备、运维、安全、高级数字人预研。

## 13. 一句话结论

这个项目最合理的预算路径是：

> 先用 100-200 万做技术验证，再用 200-500 万做 MVP；如果 MVP 证明体验成立，再投入 700-1500 万做产品化 Beta；最终要完整实现文档中的全部技术目标，建议按 1800-4000 万规划。

真正应该优先投入的不是动作生成和数字人炫技，而是：

1. 稳定人格；
2. OOC 控制；
3. 长期记忆；
4. 关系状态；
5. 语音与 Unity 表现；
6. 管理后台和测试体系。

这些才是项目能不能成立的核心。
