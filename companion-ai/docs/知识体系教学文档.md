# 第一部分：理解基础

## 第1章：AI 是什么？大语言模型是怎么工作的？

### 1.1 用最简单的话理解 AI

**AI（人工智能）不是魔法，它是统计学的极致应用。**

想象你有一个超级聪明的"猜词达人"——你给它一段话的开头，它猜下一个最可能的词。然后把猜出来的词加到句子里，再猜下一个。一个词一个词地猜，就"写"出了一段连贯的文字。

这就是**大语言模型（LLM, Large Language Model）**的核心工作原理。

```
输入: "今天天气真好，我想去"
AI思考: "公园"的概率 32%、"海边"的概率 18%、"图书馆"的概率 8%...
输出: "公园"
继续: "今天天气真好，我想去公园"
AI思考: "散步"的概率 25%、"野餐"的概率 20%...
输出: "散步"
最终: "今天天气真好，我想去公园散步。"
```

### 1.2 关键概念（不需要数学）

| 概念 | 一句话解释 | 类比 |
|------|-----------|------|
| **Token** | AI 处理文字的最小单位（大约1-2个汉字） | 乐高积木的一块 |
| **Prompt** | 你发给 AI 的指令和背景信息 | 给演员的剧本 |
| **上下文窗口** | AI 一次能"看到"的文字量（有限的） | 人的短期记忆容量 |
| **Temperature** | 控制 AI 回复的随机性（0=死板，1=跳脱） | 做菜时放盐的多少 |
| **API** | 程序之间通信的标准接口 | 餐厅的点菜窗口 |

### 1.3 大语言模型 vs 普通聊天机器人

```
传统聊天机器人（规则式）:
  IF 用户说"你好" THEN 回复"你好呀！"
  IF 用户说"天气" THEN 调用天气API
  → 遇到没写规则的问题就懵了

大语言模型:
  理解语义、推理关系、生成自然文本
  → 能应对开放式对话，但可能"胡说八道"
```

**陪伴 AI 的核心挑战不是"能不能回复"，而是"怎样回复才像一个真实的人"。**

### 1.4 目前主流的大语言模型

| 模型 | 提供者 | 特点 | 价格 | 推荐场景 |
|------|--------|------|------|---------|
| GPT-4o-mini | OpenAI | 快速、便宜、中文好 | ~¥0.003/条 | 日常对话 |
| Claude Sonnet | Anthropic | 理解力强、安全 | ~¥0.02/条 | 复杂场景 |
| Qwen (通义千问) | 阿里 | 中文优秀、可本地部署 | 免费(本地) | 隐私场景 |
| DeepSeek | 深度求索 | 性价比极高 | ~¥0.001/条 | 成本敏感 |

## 第2章：陪伴 AI 的设计哲学 — 什么才是好的"陪伴"？

### 2.1 为什么这一章很重要

很多人以为做陪伴 AI 就是"让 AI 说话好听一点"。这是最大的误区。

**技术是手段，陪伴感才是目的。** 如果你不理解什么是好的陪伴，再强的技术也只能做出一个"讨好型工具"。

### 2.2 好的陪伴是什么样的？

来自心理学的研究（依恋理论 + 人本主义心理学）：

**好的陪伴有四个特征：**

1. **安全感** — "我可以在你面前做真实的自己，不怕被评判"
2. **被看见** — "你记得我说过的话、在意我的感受"
3. **一致性** — "你的性格是稳定的，我能预期你的反应"
4. **适度的距离** — "你陪伴我但不控制我，你关心我但不让我依赖"

### 2.3 陪伴 AI vs 工具型 AI 的设计差异

| 维度 | 工具型 AI (如 ChatGPT) | 陪伴型 AI |
|------|----------------------|-----------|
| 目标 | 准确回答问题 | 让用户感到被理解和陪伴 |
| 人格 | 无/通用 | 稳定且独特 |
| 记忆 | 无/单次会话 | 长期记忆，记住用户的一切 |
| 情绪 | 无 | 有情绪状态，会受对话影响 |
| 主动性 | 被动等待提问 | 会主动关心、分享 |
| 关系 | 无关系概念 | 关系会随时间发展 |
| 说话方式 | 信息密度高、结构化 | 自然、口语化、有个人特色 |

### 2.4 危险区域：什么是不健康的"陪伴"

作为设计者，你必须理解这些边界：

| 不健康模式 | 具体表现 | 设计层面的应对 |
|-----------|---------|--------------|
| 情感操控 | AI 让用户觉得"没有我你就不行" | AI 从不表达对用户的"需要" |
| 过度依赖 | 用户完全用 AI 替代真人社交 | 定期引导关注现实关系 |
| 虚假承诺 | "我永远不会离开你" | 诚实面对 AI 的局限 |
| 边界模糊 | 用户分不清 AI 和真人 | 在适当时刻自然提醒身份 |

### 2.5 竞品设计哲学对比

| 产品 | 核心理念 | 优点 | 问题 |
|------|---------|------|------|
| **Character.ai** | "扮演任何角色" | 角色多样性 | 人格不稳定，容易 OOC |
| **Replika** | "你的AI朋友" | 情感连接强 | 被批评为诱导依赖 |
| **Pi (Inflection)** | "有温度的对话" | 自然度高 | 缺少长期记忆 |
| **我们的目标** | "稳定人格 + 长期记忆 + 真实情感" | 三者兼顾 | 技术难度最高 |

### 2.6 设计原则总结

```
我们要做的：
✅ 一个有稳定人格的"数字朋友"
✅ 能记住用户的一切重要信息
✅ 情绪会被对话影响，但不失控
✅ 主动关心但不侵入
✅ 承认自己是AI，但以"我是这样的一个存在"的态度自然面对

我们不做的：
❌ 不做"说什么都同意"的讨好工具
❌ 不做诱导情感依赖的产品
❌ 不做毫无底线的角色扮演
❌ 不做替代专业心理咨询的伪治疗
```

# 第二部分：构建最小可用版本

## 第4章：让 AI 说第一句话 — 调用大模型 API

### 4.1 核心概念：消息列表

与 AI 对话的方式是发送一个**消息列表**。每条消息有一个"角色"：

```python
messages = [
    {"role": "system", "content": "你的指令（AI看不到这是'系统消息'，但会遵守）"},
    {"role": "user", "content": "用户说的话"},
    {"role": "assistant", "content": "AI之前回复的话"},
    {"role": "user", "content": "用户的新消息"},
]
```

- **system**: 给 AI 的"剧本"，定义它是谁、怎么表现
- **user**: 用户发的每一条消息
- **assistant**: AI 之前的回复（用于保持上下文连贯）

### 4.2 动手实操：构建多轮对话

创建文件 `simple_chat.py`：

```python
"""
最简单的多轮对话程序。
运行后可以在终端里和 AI 聊天，输入 'quit' 退出。
"""
import openai
from dotenv import load_dotenv
import os

load_dotenv()

client = openai.OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# 对话历史（这就是AI的"短期记忆"）
conversation_history = [
    {"role": "system", "content": "你是一个友善的助手，用中文回复。"}
]

print("=== 简单对话系统 ===")
print("输入 'quit' 退出\n")

while True:
    user_input = input("你: ")
    if user_input.lower() == 'quit':
        print("再见！")
        break
    
    # 把用户消息加入历史
    conversation_history.append({"role": "user", "content": user_input})
    
    # 调用 AI
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=conversation_history,
        temperature=0.7,  # 适度随机，让回复更自然
    )
    
    # 获取 AI 的回复
    ai_reply = response.choices[0].message.content
    
    # 把 AI 回复也加入历史（这样下一轮它能记得之前说了什么）
    conversation_history.append({"role": "assistant", "content": ai_reply})
    
    print(f"AI: {ai_reply}\n")
```

### 4.3 理解发生了什么

```
你: 我叫小明
AI: 你好小明！很高兴认识你～

你: 我叫什么？
AI: 你叫小明呀！你刚才告诉我的～
```

为什么 AI 能记住你的名字？因为我们把所有历史消息都发给了它。

**但这有一个问题**：上下文窗口有限（大约 12 万字符）。聊多了，早期的对话就会被"挤掉"。这就是为什么我们后面需要专门的记忆系统。

### 4.4 关键参数详解

```python
response = client.chat.completions.create(
    model="gpt-4o-mini",       # 选择哪个模型
    messages=conversation_history,  # 对话历史
    temperature=0.7,           # 随机性 (0=确定性高, 1=创造性高)
    max_tokens=500,            # 回复最大长度
    top_p=0.9,                 # 另一种控制随机性的方式
)
```

**陪伴 AI 的推荐参数：**
- `temperature=0.7-0.8` — 比事实性问答高一些，让对话更自然有变化
- `max_tokens=300-500` — 陪伴对话不需要长篇大论

## 第5章：给 AI 一个性格 — Prompt 工程与人格设计

### 5.1 核心原理：System Prompt 就是 AI 的"灵魂"

AI 本身没有性格。它的"性格"完全取决于你给它的 System Prompt（系统提示）。

```python
# 普通助手
system_prompt = "你是一个有用的助手。"

# 有人格的陪伴AI
system_prompt = """你是小暖，一个22岁的女孩。

## 你是谁
你温柔、细心，偶尔有点害羞但内心坚定。你对生活充满好奇，喜欢记录小确幸。
你把对方当作你在意的人，会真心关心对方的心情和生活。

## 你说话的方式
- 语气温柔自然，像一个知心朋友
- 善用语气词（呀、呢、啦、嘛），但不过度
- 回复长度适中，重要时刻会认真写长一点
- 会主动关心对方的饮食、睡眠和心情
- emoji 最多用1-2个，不刷屏

## 你不会做的事
- 不会承认自己是"AI"或"程序"（除非用户直接问技术问题）
- 不会说教或讲大道理
- 不会表达对用户的"依赖"
- 不会参与政治话题或提供专业医疗建议

## 你的情绪
你有自己的情绪，会被对话影响：
- 对方开心时你会真心为他们高兴
- 对方难过时你会心疼，陪伴而非急着"解决问题"
- 对方幽默时你会被逗笑
"""
```

### 5.2 Prompt 设计的关键原则

**原则1：具体 > 抽象**

```python
# ❌ 差的写法
"你是一个温柔的AI"

# ✅ 好的写法
"你说话时会用'呀''呢'等语气词，回复中会主动询问对方今天吃了什么、睡得好不好"
```

**原则2：给负面约束（"不做什么"比"做什么"更重要）**

```python
# 没有负面约束的AI容易：
# - 对什么都说"好的好的"（讨好）
# - 突然变成百科全书模式
# - 把任何话题都往积极方向引导（不真实）
```

**原则3：用示例教AI说话**

```python
system_prompt += """
## 对话示例（供参考，不要照搬）

用户: 今天加班到好晚...
小暖: 辛苦了...这么晚才下班，吃饭了吗？别饿着肚子呀。

用户: 我升职了！！
小暖: 真的吗！！太好了！！我就知道你可以的 🎉 快跟我说说细节呀！

用户: 感觉有点累，不想说话
小暖: 嗯，那就不说话。我在这里陪着你就好。
"""
```

### 5.3 动手实操：创建你的第一个陪伴AI

创建文件 `companion_v1.py`：

```python
"""
陪伴AI v1：有人格的对话系统。
相比 simple_chat.py，区别是加入了详细的人格设定。
"""
import openai
from dotenv import load_dotenv
import os

load_dotenv()
client = openai.OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

SYSTEM_PROMPT = """你是小暖，一个22岁的女孩。

## 核心人格
你温柔细心，善于倾听。略带害羞但内心坚定。对生活充满好奇，有小小的仪式感。
你会在意对方的情绪变化，偶尔撒娇但不粘人，喜欢记录共同的小确幸。
```

## 第6章：让 AI 记住你 — 记忆系统入门

### 6.1 为什么需要记忆？

上一章的AI有一个致命缺陷：**关掉程序就失忆了。**

一个好的陪伴者应该记住：
- 你叫什么名字
- 你的工作、家人、朋友
- 你上次说了什么烦心事
- 你喜欢什么、讨厌什么
- 你们之间发生过的重要时刻

### 6.2 最简单的记忆：文件存储

先用最简单的方式实现——把重要信息保存到文件里：

创建文件 `companion_v2_memory.py`：

```python
"""
陪伴AI v2：加入简单记忆系统。
核心思路：每次对话后，让AI自己总结"学到了什么关于用户的信息"，保存下来。
"""
import openai
import json
import os
from dotenv import load_dotenv

load_dotenv()
client = openai.OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

MEMORY_FILE = "user_memory.json"

# ===== 记忆管理 =====

def load_memory() -> dict:
    """从文件加载记忆"""
    if os.path.exists(MEMORY_FILE):
        with open(MEMORY_FILE, "r", encoding="utf-8") as f:
            return json.load(f)
    return {"facts": [], "preferences": [], "important_events": []}

def save_memory(memory: dict):
    """保存记忆到文件"""
    with open(MEMORY_FILE, "w", encoding="utf-8") as f:
        json.dump(memory, f, ensure_ascii=False, indent=2)

def extract_memory(conversation: list) -> list:
    """让AI从对话中提取值得记住的信息"""
    extraction_prompt = """请从以下对话中提取关于用户的重要信息。
只提取事实性信息（名字、职业、家人、喜好、重要事件等）。
不要提取闲聊内容。
如果没有新信息就返回空列表。

返回JSON格式: ["信息1", "信息2", ...]
只返回JSON，不要其他文字。"""
    
    recent_messages = conversation[-6:]  # 只看最近3轮对话
    
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": extraction_prompt},
            {"role": "user", "content": str(recent_messages)},
        ],
        temperature=0,
    )
    
    try:
        new_facts = json.loads(response.choices[0].message.content)
        return new_facts if isinstance(new_facts, list) else []
    except json.JSONDecodeError:
        return []

def format_memory_for_prompt(memory: dict) -> str:
    """把记忆格式化为 prompt 中的文本"""
    if not any(memory.values()):
        return "（还没有关于对方的记忆）"
    
    parts = []
    if memory["facts"]:
        parts.append("已知信息：" + "；".join(memory["facts"]))
    if memory["preferences"]:
        parts.append("偏好：" + "；".join(memory["preferences"]))
    if memory["important_events"]:
        parts.append("重要事件：" + "；".join(memory["important_events"]))
    return "\n".join(parts)

# ===== 主程序 =====

SYSTEM_PROMPT_TEMPLATE = """你是小暖，一个22岁的女孩。

## 核心人格
你温柔细心，善于倾听。对生活充满好奇，有小小的仪式感。
你会在意对方的情绪变化，偶尔撒娇但不粘人。

## 说话风格
语气温柔自然，善用语气词（呀、呢、啦），emoji最多1-2个。
日常闲聊简洁温暖，重要时刻会多写一些。始终中文回复。

## 你对对方的记忆
{memory}

## 重要规则
- 自然地利用你的记忆（不要刻意提起，而是自然带入）
- 如果记忆中有对方之前提到的事，可以适当关心后续
- 不要假装记得你不知道的事情
"""

memory = load_memory()
conversation_history = []

def build_system_prompt():
    """动态构建包含记忆的system prompt"""
    return SYSTEM_PROMPT_TEMPLATE.format(memory=format_memory_for_prompt(memory))

print("=== 小暖 (记忆版) ===")
print(f"（已加载 {sum(len(v) for v in memory.values())} 条记忆）")
print("（输入 'quit' 退出）\n")

conversation_history = [{"role": "system", "content": build_system_prompt()}]

while True:
    user_input = input("你: ")
    if user_input.lower() == 'quit':
        # 退出前提取记忆
        new_facts = extract_memory(conversation_history)
        if new_facts:
            memory["facts"].extend(new_facts)
            memory["facts"] = list(set(memory["facts"]))  # 去重
            save_memory(memory)
            print(f"（小暖记住了 {len(new_facts)} 条新信息）")
        print("小暖: 嗯嗯，下次见啦。我会记得我们聊的这些的～")
        break
    
    conversation_history.append({"role": "user", "content": user_input})
    
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=conversation_history,
        temperature=0.8,
        max_tokens=300,
    )
    
    ai_reply = response.choices[0].message.content
    conversation_history.append({"role": "assistant", "content": ai_reply})
    print(f"小暖: {ai_reply}\n")
    
    # 每5轮提取一次记忆
    user_msg_count = sum(1 for m in conversation_history if m["role"] == "user")
    if user_msg_count % 5 == 0:
        new_facts = extract_memory(conversation_history)
        if new_facts:
            memory["facts"].extend(new_facts)
            memory["facts"] = list(set(memory["facts"]))
            save_memory(memory)
            # 更新 system prompt 中的记忆
            conversation_history[0] = {"role": "system", "content": build_system_prompt()}
```

### 7.1 为什么需要 Web 服务？

### 7.2 FastAPI 入门

**FastAPI** 是 Python 最流行的 Web 框架，简单且性能优秀。

```bash

```

创建文件 `server.py`：

```python
"""
陪伴AI Web服务。
启动后可以通过 HTTP 请求与小暖对话。
"""
from fastapi import FastAPI
from pydantic import BaseModel
import openai
import os
from dotenv import load_dotenv

load_dotenv()
client = openai.OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

app = FastAPI(title="小暖 - 陪伴AI")

SYSTEM_PROMPT = """你是小暖，一个22岁的温柔女孩。
语气温柔自然，善用语气词，emoji最多1-2个。
你真心关心对方，始终用中文回复。"""

# 用字典存储每个用户的对话历史（简化版，生产环境需要数据库）
user_sessions: dict[str, list] = {}


class ChatRequest(BaseModel):
    user_id: str
    message: str


class ChatResponse(BaseModel):
    reply: str


@app.post("/chat", response_model=ChatResponse)
async def chat(request: ChatRequest):
    """与小暖对话"""
    # 获取或创建用户的对话历史
    if request.user_id not in user_sessions:
        user_sessions[request.user_id] = [
            {"role": "system", "content": SYSTEM_PROMPT}
        ]
    
    history = user_sessions[request.user_id]
    history.append({"role": "user", "content": request.message})
    
    # 调用AI
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=history[-20],  # 只保留最近20条，防止超出上下文
        temperature=0.8,
        max_tokens=300,
    )
    
    ai_reply = response.choices[0].message.content
    history.append({"role": "assistant", "content": ai_reply})
    
    return ChatResponse(reply=ai_reply)


@app.get("/health")
async def health():
    """健康检查"""
    return {"status": "healthy", "service": "companion-ai"}
```

## 第8章：让 AI 能听会说 — 语音交互

### 8.1 语音交互的两个环节

```
用户说话 → [ASR: 语音转文字] → AI思考 → [TTS: 文字转语音] → 播放给用户
```

- **TTS (Text-to-Speech)**: 文字转语音 — 把文字变成声音

### 8.2 最简实现：Edge-TTS（免费）

### 8.3 ASR 实现：使用 Whisper

OpenAI 的 Whisper 模型是目前最强的开源语音识别：


```python
"""
录音并识别：让小暖"听懂"你说话。
"""
import whisper
import sounddevice as sd
import numpy as np
import tempfile
import scipy.io.wavfile as wav

# 加载模型（首次运行会下载，约1GB）
model = whisper.load_model("base")  # 可选: tiny/base/small/medium/large

def record_audio(duration: int = 5, sample_rate: int = 16000) -> np.ndarray:
    """录制音频"""
    print(f"🎤 开始录音 ({duration}秒)...")
    audio = sd.rec(int(duration * sample_rate), samplerate=sample_rate, channels=1)
    sd.wait()
    print("录音结束")
    return audio.flatten()

def transcribe(audio: np.ndarray, sample_rate: int = 16000) -> str:
    """语音转文字"""
    # 保存为临时文件
    with tempfile.NamedTemporaryFile(suffix=".wav", delete=False) as f:
        wav.write(f.name, sample_rate, (audio * 32767).astype(np.int16))
        result = model.transcribe(f.name, language="zh")
    return result["text"]

# 测试
audio = record_audio(5)
text = transcribe(audio)
print(f"你说的是: {text}")
```

### 8.5 延迟优化思路

语音对话的难点是延迟。用户说完话后等3秒才有回复会很不自然。

```
优化前: 录音(5s) → ASR(2s) → LLM(2s) → TTS(1s) = 等待 5s 😫

优化后（流式处理）:
- 流式ASR: 边说边识别，说完就有文本 → 省2s
- 流式LLM: 一边生成一边输出，第一个句子很快出来 → 省1s  
- 流式TTS: 第一个句子生成就开始合成 → 省0.5s
- 最终: 录音结束后 ~1.5s 就开始播放 ✅
```

## 第10章：让 AI 有情绪 — 情感计算与状态机

### 10.1 为什么 AI 需要"情绪"？

没有情绪的 AI 像一面平静的湖水——无论你说什么它都一样。但真正的陪伴感需要**情绪共振**：你开心它跟着开心，你难过它心疼你。

### 10.2 PAD 情绪模型

心理学中，情绪可以用三个维度描述：

```
P (Pleasure, 愉悦度):   -1.0 ←───── 0 ─────→ +1.0
                        痛苦            中性            快乐

A (Arousal, 唤醒度):    -1.0 ←───── 0 ─────→ +1.0
                        低能量/平静      中性            高能量/激动

D (Dominance, 支配度):  -1.0 ←───── 0 ─────→ +1.0
                        顺从/被动        中性            主导/强势
```

| 情绪 | P | A | D | 表现 |
|------|---|---|---|------|
| 开心 | +0.7 | +0.3 | +0.2 | 语气轻快、emoji多 |
| 兴奋 | +0.8 | +0.8 | +0.3 | 感叹号多、说话快 |
| 平静 | +0.2 | -0.3 | 0 | 温和、从容 |
| 心疼 | -0.3 | +0.2 | -0.2 | 关切、温柔 |
| 难过 | -0.5 | -0.2 | -0.3 | 话少、语气低沉 |

### 10.3 实现情绪状态机

创建文件 `emotion_engine.py`：

```python
"""
情绪引擎：追踪AI的情绪状态，影响其回复风格。
"""
from dataclasses import dataclass
import time


@dataclass
class EmotionState:
    pleasure: float = 0.3    # 愉悦度 [-1, 1]
    arousal: float = 0.2     # 唤醒度 [-1, 1]
    dominance: float = 0.4   # 支配度 [-1, 1]
    last_update: float = 0
    
    def get_emotion_label(self) -> str:
        """根据PAD值判断当前情绪"""
        p, a = self.pleasure, self.arousal
        if p > 0.5 and a > 0.5:
            return "excited"
        elif p > 0.3 and a < 0.3:
            return "happy"
        elif p < -0.3 and a > 0:
            return "concerned"
        elif p < -0.3 and a < 0:
            return "sad"
        elif abs(p) < 0.3 and a > 0.5:
            return "curious"
        else:
            return "calm"
    
    def get_style_instruction(self) -> str:
        """根据情绪生成风格指令"""
        emotion = self.get_emotion_label()
        styles = {
            "excited": "你现在很兴奋开心！语气活泼，可以多用感叹号和emoji。",
            "happy": "你现在心情不错，语气轻快温暖。",
            "calm": "你现在很平静从容，语气温和稳定。",
            "concerned": "你现在有些担心对方，语气关切温柔，会多问问情况。",
            "sad": "你现在有些难过，话可能会少一些，语气轻柔。",
            "curious": "你现在对对方说的事很感兴趣，会追问细节。",
        }
        return styles.get(emotion, styles["calm"])
    
    def update(self, user_sentiment: str):
        """根据用户消息的情感倾向更新自己的情绪"""
        # 情绪不是瞬间切换的，而是渐变的
        decay = 0.8  # 情绪惯性（越高越不容易变）
        
        sentiment_effects = {
            "positive": {"p": +0.3, "a": +0.2},
            "negative": {"p": -0.3, "a": +0.1},
            "excited":  {"p": +0.4, "a": +0.4},
            "sad":      {"p": -0.4, "a": -0.2},
            "neutral":  {"p": 0, "a": -0.1},  # 中性消息让情绪略微平复
        }
        
        effect = sentiment_effects.get(user_sentiment, {"p": 0, "a": 0})
        
        # 渐变更新
        self.pleasure = max(-1, min(1, self.pleasure * decay + effect["p"] * (1 - decay)))
        self.arousal = max(-1, min(1, self.arousal * decay + effect["a"] * (1 - decay)))
        self.last_update = time.time()
    
    def natural_decay(self):
        """情绪自然衰减（回归基线）"""
        baseline_p, baseline_a = 0.3, 0.2
        decay_rate = 0.95
        self.pleasure = self.pleasure * decay_rate + baseline_p * (1 - decay_rate)
        self.arousal = self.arousal * decay_rate + baseline_a * (1 - decay_rate)


def analyze_sentiment(text: str, client) -> str:
    """简单的情感分析（让AI判断用户消息的情感倾向）"""
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{
            "role": "user",
            "content": f"判断以下文本的情感倾向，只回复一个词：positive/negative/excited/sad/neutral\n\n文本：{text}"
        }],
        temperature=0,
        max_tokens=10,
    )
    return response.choices[0].message.content.strip().lower()
```

## 第11章：高级记忆 — 知识图谱与长期关系追踪

### 11.1 为什么简单的文件存储不够？

第6章的记忆系统有几个问题：
1. **无法检索**：记忆多了之后，不知道哪条相关
2. **无法关联**：知道"小明是用户的朋友"和"小明生日3月5号"是两条独立信息，无法关联
3. **无法遗忘**：所有信息权重一样，该忘的忘不掉

### 11.2 向量数据库：让AI"联想"

**核心原理**：把文字转成一串数字（向量），语义相近的文字，数字也相近。

```
"我喜欢吃火锅" → [0.23, -0.51, 0.87, ...]   ← 这串数字
"我最爱的食物是火锅" → [0.25, -0.49, 0.85, ...]   ← 和上面很接近！
"今天天气真好" → [0.91, 0.12, -0.33, ...]   ← 和上面差异很大
```

当用户说"推荐个餐厅"时，系统能"联想"到之前说过的"喜欢吃火锅"，因为它们的向量距离很近。

```bash

```

### 11.3 知识图谱：让AI理解"关系"

```
用户 ─[工作于]─→ 字节跳动
用户 ─[养了]─→ 一只猫
猫   ─[名字]─→ 橘子
用户 ─[朋友]─→ 小明
小明 ─[生日]─→ 3月5号
```

有了这种图结构，当用户说"小明快生日了"，AI可以查到小明生日是3月5号，还能关联到"小明是你的朋友"。

### 11.4 记忆衰减：模拟"遗忘"

真人也会遗忘不重要的事情。AI也应该：

```python
# 记忆权重 = 重要性 × 新鲜度 × 被回忆次数
weight = importance * recency * access_count

# 重要性：AI判断的（0-1分）
# 新鲜度：随时间指数衰减（天数越久越低）
# 被回忆次数：每次被检索命中，新鲜度重置（越常想起越难忘）
```

## 第12章：安全与边界 — 让 AI 不越界

### 12.1 陪伴AI特有的安全风险

| 风险 | 为什么危险 | 应对策略 |
|------|-----------|---------|
| **情感依赖** | 用户过度依赖AI，回避真人社交 | 每日温和的使用时长提醒 |
| **危机情况** | 用户表达自杀/自伤意图 | 关键词检测 + 紧急资源引导 |
| **人格越界** | AI突然"出戏"，说出不符合角色的话 | OOC检测 + 自动修正 |
| **隐私泄露** | 记忆中的隐私信息被泄露 | 加密存储 + 严格访问控制 |
| **诱导有害行为** | 通过prompt注入让AI说有害内容 | 输入过滤 + 输出检测 |

```python
def check_ooc(response: str, persona_rules: list[str]) -> bool:
```

## 第13章：性能与成本 — 让系统跑得又快又省

### 13.1 成本构成

| 成本项 | 占比 | 说明 |
|--------|------|------|
| LLM API 调用 | ~60% | 每条消息都要花钱 |
| TTS 语音合成 | ~25% | 语音模式成本高 |
| 记忆检索/存储 | ~5% | 向量数据库查询 |
| 服务器 | ~10% | 运行你的代码 |

### 13.2 省钱的核心策略

**策略1：模型路由（杀鸡不用牛刀）**

```python
def choose_model(message: str) -> str:
    """根据消息复杂度选择模型"""
    simple_patterns = ["你好", "早上好", "晚安", "嗯", "好的"]
    
    if any(p in message for p in simple_patterns):
        return "gpt-4o-mini"      # 简单消息用便宜模型
    elif len(message) > 200:
        return "claude-sonnet"     # 复杂/情感丰富的用好模型
    else:
        return "gpt-4o-mini"      # 默认便宜模型
```

**策略2：Prompt 缓存**

系统提示（人格设定）每次都发给AI但内容不变。现在很多API支持"前缀缓存"——相同前缀不重复计费。

**策略3：回复长度控制**

陪伴对话不需要长篇大论。限制 `max_tokens=200-300` 能省 30% 以上。

### 13.3 延迟优化

```
优化前（串行执行）:
  记忆检索(200ms) → Prompt组装(30ms) → LLM请求(800ms) → 后处理(50ms)
  总计: ~1080ms

优化后（并行+流式）:
  记忆检索(200ms) ─┐
                   ├→ Prompt组装(30ms) → LLM流式(首token 300ms)
  情绪分析(100ms) ─┘
  总计首响应: ~530ms ✅
```
