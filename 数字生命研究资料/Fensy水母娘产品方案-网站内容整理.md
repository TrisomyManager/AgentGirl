## 一、项目总览

### 1.1 定位

**水母娘「Fensy」** — 以**长期关系成长**为核心的多端数字生命产品启动方案。

- **手机端**：高频陪伴与关系留存。  
- **设备端**：轻量、常伴随触达。  
- **车机与 IoT**：把数字生命带入真实生活。  
- **线下 MR**：高光剧情与关系升级。  

目标感知：Fensy 不是「会说话的角色」，而是**会成长、会回应、会记住你**的数字生命。

### 1.2 多端形态摘要

| 形态 | 要点 |
| --- | --- |
| 手机端主应用 | 高频入口、日常聊天、关系成长、成就、碎片收集、剧情解锁 |
| 设备端陪伴 | 低门槛触达、情绪反馈、提醒陪伴、物理存在感、与手机状态联动 |
| 车机 / 出行 | 通勤助手、路线建议、导航联动、途中主动提醒、车家状态接力 |
| 微信 / 小程序 / 内容渠道 | 轻量接入、活动承接、预约导流、公域触达、私域沉淀 |
| 线下 MR 高光 | 首次惊艳、关系升级节点、沉浸互动、CG 演出与高光补偿 |

### 1.3 新版产品方案口径（三主线）

主站已收敛为：

1. **房间首页** — 首页即房间，自然语言 + AI 生成角色动画 + 记忆回流，建立日常在场感。  
2. **剧情副本** — 关卡式副本；视频、立绘、语音对白、互动选择演绎主线 / 支线 / 角色回忆。  
3. **跨端生活介入** — 自 App 延伸至桌面、车机、音箱、耳机、手表、MR、IoT，在**授权边界**内介入真实生活。

### 2.1 角色档案（三方向对比）

| 属性 | 方向 A：深海之灵 | 方向 B：易碎微光 | 方向 C：织梦歌者 |
| --- | --- | --- | --- |
| 姓名 | Fensy（小汐） | 同左 | 同左 |
| 种族 | 水母族少女 | 同左 | 同左 |
| 年龄 | 20 岁 | 同左 | 同左 |
| 统一内核 | 治愈、温柔、安静有力量 | 同左 | 同左 |
| 身份表达 | 梦境边缘的灯塔型守护者 | 吞噬坏情绪的治愈者 | 编织梦境的舞台系歌者 |
| 关系方式 | 靠时间积累与稳定陪伴建立依赖 | 靠情绪波动与双向救赎建立羁绊 | 靠视觉新鲜感与仪式感维持吸引 |
| 语言风格 | 轻、慢、安抚感强 | 热烈、试探、容易显露不安 | 爱分享、爱展示、富有舞台感 |
| 典型状态 | 像潮汐一样安静地陪在旁边 | 主动靠近又害怕你走开 | 把每次见面都做成小型梦境秀 |

### 2.2 名称意境

- **芬**：清芬、草木之香 — 温柔内敛、干净有灵气。  
- **汐**：夜潮、潮汐起落 — 安静、有节律、诗意、从容通透。  
- **合意**：小汐 = 清芬伴潮汐，温柔自安然（晚风携花香、夜色海潮；柔美不娇弱，清新不俗气）。

### 2.3 世界观与核心意象

- Fensy 来自水母族，生活在**深海与梦境交界处**；以人类情绪波动为食，只对**真诚情感**共鸣。  
- **核心意象**：海洋（情感的浩瀚）、水母（柔软透明需温柔对待）、光点（记忆与情绪可视化）、梦境（与现实平行的情感空间）。  
- **统一角色目标**：接住用户难以安放的情绪，把「被陪伴」变成可感知的体验。

### 2.4 三方向一句话摘要

- **A 深海之灵**：像一盏永远亮着的灯 — 不打扰但持续在场，陪伴即安全感。  
- **B 易碎微光**：会撒娇也会逞强的治愈者 — 需要回应，用情绪反差强化连接。  
- **C 织梦歌者**：梦境秀场导演 — 用视听与服装变化制造惊喜，把情绪转成审美体验。

## 三、设计原则

**页面**：[设计原则](https://fensy-ip-report.pages.dev/principles/index.html)

### 3.1 调研参考（站内列出）

- 女性 IP 角色设计调研报告  
- 基于头部 IP 的深度解构报告  
- 人与虚拟人亲密关系研究报告  

### 3.2 七条核心体验原则

1. **MR 互动仅发生在线下体验店** — 高沉浸空间互动、关系推进主剧情、家园共处均在线下完成。  
2. **线上只承担关系延续** — 聊天、轻互动、记忆回调、情绪陪伴、预约下次见面；不替代线下 MR。  
3. **对话必须发生在互动里** — 一起看、一起做、一起选、一起回忆中自然产生，非机械问答。  
4. **场景基于 Fensy 的家园系统** — 主要互动集中在房间与家园，形成强记忆点。  
5. **选择必须有后续** — 每个回答进入追问、行为反馈或关系变化。  
6. **关系必须可成长** — 同一问题在不同亲密阶段问法、语气、深度不同。  
7. **记忆必须可回调** — 引用用户过去线下与线上的选择与回答，关系有连续性。

### 3.3 产品边界

**线下 MR 体验店适合**：第一次见面、关键关系阶段推进、家园共同活动、高情绪价值互动、默契测试与双向选择、需空间定位的互动等。

**线上应用适合**：日常聊天、简单问候、情绪陪伴、记忆回顾、预约下次见面；**不适合**替代 MR 沉浸体验。

### 3.4 线上应用承接设计

**可承接**：延续线下话题与近况、主动问候、记忆回调、轻量选择互动、预约感营造、家园轻养成反馈（装饰 / 礼物 / 纪念物变化等）。

**不应替代**：首次见面剧情、关键关系升级、高沉浸双人共处、空间化仪式感事件、需空间定位与手势 / 视线交互的玩法。

**线上存在的意义**：两次线下之间保持情感连接、积累对下次见面的期待、轻量陪伴与提醒、让记忆跨场景延续 — 而非「低配版 Fensy」。

## 四、应用场景

**页面**：[应用场景](https://fensy-ip-report.pages.dev/interaction/home.html)

### 4.1 从「多端展示」到「连续生活路径」

- **手机房间**：日常留存。  
- **剧情副本**：内容消费与关系推进。  
- **跨端生活**：长期壁垒与品牌想象力。

### 4.2 场景一：移动端首页（房间）

- 打开 App 即进入房间，**角色一直在场**；待机、回应、走近、害羞、安抚等由 **AI 生成动画** + 自然语言交流支撑。  
- **首页状态**：默认房间 + 角色，非传统功能宫格；可见角色在做什么、今日情绪、房间变化。  
- **自然交流**：语音 + 文字；对话驱动表情、视线、姿态、灯光与小物件反馈。  
- **AI 动画路线**：动作库兜底 + 条件生成 + 情绪标签 + 亲密度修正。  
- **房间养成**：回忆、礼物、剧情解锁物、偏好 — 关系进展的可视化。  
- **入口**：「剧情」进副本；「生活」进跨端能力面板。

### 4.3 场景二：剧情模块（关卡化副本）

- 参考「关卡地图」组织主线、支线、角色回忆、限时事件；**视频 + 立绘 + 语音 + 选择**，奖励**回流房间**（摆件、待机动作、称呼、主动问候等）。  
- **内容形态**：主线关卡、角色回忆（短内容）、日常事件、限时副本（运营）。  
- **单关节奏建议**：开场视频 → 立绘对白 → 互动选择 → 高光演出 → 奖励回流。  
- **生产管线**：视频段落、立绘状态、表情包、动作片段、对白脚本、分支标签、奖励配置等可复用单元。

**示例关卡名（原站示例）**：潮汐回廊、初遇房间、海雾来信、深海异常、共鸣之夜等。

### 4.4 场景三：跨端生活介入

- **终端**：手机主控；桌面工作陪伴；车机出行；音箱 / 耳机语音在场；手表轻提醒；MR 高光沉浸。  
- **IoT**：米家、Matter/HomeKit、鸿蒙智联、车机等；一期优先灯光、音响、空气、窗帘、场景开关，不求一次全接入。  
- **行动边界**：建议 / 待确认执行 / 自动执行分档；用户不失去控制感。  
- **关系表达**：调暗灯、提醒带伞等需服务于情感叙事（例如记得你讨厌淋雨）。

### 4.5 使用路径与统一底座

**典型路径**：进入房间 → 自然对话 → 进入剧情 → 奖励回流 + 跨端授权延续陪伴。

**共用底座**：人格、记忆、情绪、亲密度、动作表现、权限系统 — 非三套割裂功能。

### 4.6 家园系统定位

家园不删除，作为**长期母空间**：承接手机房间、剧情回流、MR 高光、跨端记忆沉淀；含入口接待区、客厅、窗边、音乐角、回忆廊道、未来计划角等；光影、粒子、海洋房间、环境声为视觉底色。

## 五、技术实现（摘要）

**页面**：[技术实现](https://fensy-ip-report.pages.dev/tech/index.html)  
**口径**：Room + Story + Cross-device；房间日常在场、剧情演绎与关系推进、跨端 IoT 联动。

### 5.1 原则与一期目标

- **三模块同源**：房间、剧情、跨端共享人格、记忆、关系与授权状态。  
- **云端主脑**：核心智能、关系推进、记忆读取以云端为主；本地侧重实时表现。  
- **渐进迭代**：先跑通「能说、能演、能记住」，再扩展复杂能力。  
- **推荐组合**：Unity 统一运行时（手机 + MR）；云端编排（LLM、关系、记忆）；复用现有记忆库 / 数字生命引擎；表现先走**受控生成**（动作库、表情规则、LookAt、停顿、手势）再扩展生成式身体表现。

**一期先做**：流式 ASR/TTS、对话编排器、角色状态层、记忆调用、表情动作驱动、弱网降级。  
**后做**：全身生成动画、复杂布料、端侧大模型、本地长期记忆库、动态场景生成等。

**一期成功标准（原站表述）**：用户能感到她认真回应、记住自己，且说话时**不只有嘴在动**。

### 5.2 十层架构（原站结构）

```
客户端表现层 → 交互输入层 → 角色驱动层 → 对话与智能层 → 情绪与关系层
  → 记忆系统接入层 → 内容与配置层 → 服务端编排层 → 数据存储层 → 日志监控层
```

各层职责、输入输出及依赖关系详见原站表格（技术页「系统架构与核心模块」）。

### 5.3 核心模块与端云分工（要点）

- **必须云端**：LLM、编排、关系、记忆、日志等。  
- **必须本地**：动画、表情、口型、LookAt、Idle 等。  
- **ASR/TTS**：云端为主，端侧可作保底。  
- **关键模块清单**：ASR、TTS、LLM、对话编排器、角色意图/状态/情绪/关系、记忆接入、动作表情口型 LookAt、Idle、场景上下文、配置与埋点等（详见原站表格）。

### 5.4 主交互链路（语音）

用户语音 → VAD → 流式 ASR → 编排器（场景 + 关系 + 记忆）→ LLM（语义、情绪、行为槽位）→ 流式 TTS → 客户端口型 / 表情 / LookAt / 手势 / 播放。

文本输入跳过 ASR，其余链路一致，**禁止**「只有字幕、角色无表现」的断裂。

### 5.5 记忆与降级（要点）

- **记忆**：对话前轻量读取；对话中按需追加；对话后异步写回；控制延迟。  
- **降级**：ASR 超时引导文本；TTS 失败先文本 + 短音效；记忆超时用缓存称呼与关系；LLM 超时安全模板 + 情绪表现保底。

### 5.6 跨端 Action Gateway

- 统一抽象「提醒、播放、灯光、窗帘、场景」等为 **Action**，经网关适配米家、Matter/HomeKit、鸿蒙智联、车机、桌面 / 穿戴 / MR 等。  
- **职责**：设备注册、动作抽象、权限判断、执行回执、日志回放。  
- **授权三档**：只建议 / 待确认 / 可自动（用户明确低风险例行）；权限可在手机房间查看、暂停、撤销。

### 5.7 角色表现与高保真渲染（提纲）

- **语义到表现**：行为槽位 → 动作池、表情、视线；回复前中后的表现组织；TTS 韵律驱动表情与手势。  
- **防机械感**：池化与权重随机、Idle 与说话分层、亲密度参与权重等。  
- **动画路线**：动捕 / 面捕 + 精修 + 标准动作库 + 条件生成；V1 建议「检索增强 + 条件生成 + 动作库回退」。  
- **情绪标注**：Russell 环形模型（效价 × 唤醒度）+ 亲密度阶段修正表（原站含详细表格）。  
- **渲染五维**：多层眼球、表情修正与情绪材质、发片到发丝分级、布料物理与神经加速、皮肤 SSS 与环境融合与水母荧光等；**六层状态机**联动（语义 / 面部 / 眼神 / 身体 / 材质 / 物理）。  
- **面部基准建议**：以 MetaHuman Facial Description Standard 为主基准，ARKit 映射互通，FACS/AU 作标注语言。

### 5.8 平台与性能预算（摘录）

- 手机目标 **60fps**（最低可接受约 **45fps**）；MR 基线 **72fps**，高端向 **90fps**。  
- 语音首字延迟理想 **0.8–1.5s**，超过 **2s** 需思考 / 等待反馈；文本回复 **1.5s** 内为佳，**3s+** 需过渡态。  
- Vision Pro：原站说明**不作为首发主目标**，可作后续 MR 高端对照研究项。

### 5.9 开发阶段（P0–P3）

| 阶段 | 目标摘要 |
| --- | --- |
| P0 技术验证 | Unity 与编排最小链路；ASR/LLM/TTS/口型/表情连通；记忆欢迎语实验 |
| P1 最小可用 | 手机稳定会话闭环；MR 单场景对话与存在感；关系 / 情绪 / 记忆写回 |
| P2 高质量演示 | 动作池与 Idle；打断 / 追问 / 弱网降级与排障；强化「被记住感」欢迎 |
| P3 可上线 | 监控、灰度、日志回放与热更新；完整性能档位；运营参数与发布流程 |

**技术选型表（原站）**：Unity 2022.3 LTS、Animator + Timeline + 事件层、关卡 + 视频 CDN + 资源包、Whisper 类 / 商业 ASR、带情绪参数 TTS、云端 LLM、Python/FastAPI + WebSocket/SSE、复用记忆库、Action Gateway + 厂商适配器、结构化埋点 + Trace 等。
